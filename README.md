# 中国编年史速查表

原生微信小程序原型，用于快速检索中国历史朝代、人物、官职权限与关键事件。

## 功能

- 朝代顺序骨架：夏朝到清朝共 19 组历史阶段
- 朝代标签释义：61 个资料状态、制度、文化、事件和时期标签均可点击查看定义、朝代语境与认知提示
- 东周拆分：春秋时期、战国时期已显式展示，并接入高频人物和事件原型
- 秦朝样板库：人物、时期、事件、官职权限、争议观点
- 人物详情页：基础档案、简历式履历、相关事件、不同认知观点
- 人物关系图谱：在人物详情页查看君臣、师承、政敌、同盟、战场对峙等关系，并可跳转到关联人物
- 人物头像：原有 575 位深度人物均已接入 ComfyUI 生成头像；新增教材通史人物已按帝王、武将、文官、其他四类生成时代服饰提示词，并采用轻量文字肖像占位
- 简历式履历：每条履历包含名义职责、管辖权限、实际影响力和职能近似
- 事件详情页：背景、经过、结果、长期影响和相关人物
- 五代十国标注：每个国家显示“五代十国”标签，并标注并存或跨越的中原政权
- 搜索：支持人物、事件、官职、朝代标签和关键词

## 性能与包体策略

- 首页按当前模式加载：朝代、人物、事件三个视图只计算当前展示所需数据，避免一次性装饰全量人物和事件
- 朝代详情懒展开：默认只展开秦朝和五代十国，其余朝代在展开或命中搜索时再生成时期、人物、帝王年表和事件列表
- 长列表分页渲染：人物、帝王、事件索引默认每次显示 40 条，通过“加载更多”继续展开，降低初次切换页签的渲染压力
- 帝王资料分层：首页列表只向视图层传递检索和展示所需字段，进入详情页后再加载完整人物志，避免大段履历和提示词进入首页 `setData`
- 人物详情分包：原有完整人物志与头像位于 `person-package`，新增 515 位教材通史人物和事件详情位于 `curriculum-package`；首页只保留轻量检索索引
- 头像资产分层：45 位重点人物在主包保留 64px 缩略图，原有 575 位人物的详情图统一放入人物分包；新增人物先使用零图片增量的文字肖像，避免头像使分包超过 2 MiB
- 内容补齐策略：君主长文使用结构化字段与逐人史实数据组合，不新增图片资源；史料不足时明确说明证据边界，不虚构童年、性格或政策细节
- 扩展补丁策略：扩展层支持给已有基础人物和事件追加关联，避免为了补关系反复改动基础样板数据
- 生成源图隔离：`assets/avatar-sources/` 已从小程序项目打包范围排除，只保留压缩后的运行头像
- 开发文件隔离：`scripts/`、`workflows/`、源图和项目说明文件不进入运行包；本地审计为主包约 1.73 MiB、人物分包约 1.75 MiB、教材通史分包约 0.03 MiB
- 后续建议：继续扩展百科式长文时，优先把内容按朝代拆分为数据分包或云端接口，保持首屏加载稳定

全量内容、关系、姓名搜索、加载性能和包体检查可运行：

```bash
node scripts/audit-content-quality.js
node scripts/audit-avatar-coverage.js
```

## 界面结构

- 首页采用紧凑通史标题栏、统计摘要、吸顶搜索和朝代/人物/事件分段切换，朝代卡片按需展开
- 人物页采用头像档案头、吸顶五页签、简历式履历时间线、双列关系图谱和关联事件列表
- 事件页采用事件档案头、脉络/人物/观点三页签，并按背景、经过、结果、影响形成四步时间线
- 全局视觉使用中性灰白底色，玉石绿用于当前状态，朱砂红用于年代和风险信息，青铜色用于历史分类

## 头像生成

头像提示词位于 `data/avatarPrompts.js`。可运行：

```bash
node scripts/export-avatar-prompts.js
```

批量导出所有人物的 ComfyUI 提示词。源图生成后放到：

```text
assets/avatar-sources/{人物id}.png
```

风格提示按帝王、武将、文官、其他四类区分，并叠加春秋、战国、秦、秦末汉初等时代服饰约束，整体采用“中国传统历史策略游戏感、写实厚涂、半身肖像、戏剧光影”的通用描述，不直接复刻任何商业游戏的专有角色形象。

如果本机 ComfyUI 已在 `127.0.0.1:8188` 运行，可以直接生成源图并压缩到两级运行目录。日常命令默认处理重点人物；全量生成或重新压缩 575 位人物时增加 `--all`：

```bash
node scripts/generate-avatars-comfyui.js --all
node scripts/optimize-avatar-assets.js --all
node scripts/audit-avatar-coverage.js
```

常用参数：

```bash
node scripts/generate-avatars-comfyui.js --only zhao-gao
node scripts/generate-avatars-comfyui.js --limit 3
node scripts/generate-avatars-comfyui.js --all --limit 3
node scripts/generate-avatars-comfyui.js --overwrite --checkpoint blue_pencil-XL-v7.0.0.safetensors
node scripts/generate-avatars-comfyui.js --overwrite --timeout-minutes 30
node scripts/generate-avatars-comfyui.js --overwrite --only zi-ying --seed-base 20260913
node scripts/optimize-avatar-assets.js --dry-run
```

头像发布压缩分为三档：重点人物首页缩略图最长边 64px、质量 35；重点人物详情图最长边 176px、质量 55；其余原有人物详情图最长边 96px、质量 45。可通过对应 `AVATAR_*` 环境变量临时调整，不要把源 PNG 放入小程序运行包。

如果 ComfyUI 报 `Error while deserializing header: incomplete metadata, file not fully covered`，说明 checkpoint 文件损坏或未下载完整。可运行：

```bash
node scripts/check-comfyui-checkpoints.js
node scripts/check-comfyui-checkpoints.js --quarantine
```

第二条命令会把坏模型移出 `models/checkpoints`，避免 ComfyUI 和批量脚本继续误选。

当前本机 ComfyUI 状态记录见 `COMFYUI_STATUS.md`。
